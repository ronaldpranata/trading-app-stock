# Professional Integration Testing

Unit tests check if *functions* work. Integration tests check if *systems* work. Use this guide to master integration testing in React.

## 1. Testing Custom Hooks with Redux

Testing hooks that interact with Redux (like `useStock`) requires a "Provider Wrapper".

### The Challenge
A hook like `useStock` uses `useDispatch` and `useSelector`. If you just run `renderHook(() => useStock())`, it will crash because there is no Redux Context.

### The Solution: `wrapper`
We create a temporary Redux store for the test and wrap the hook execution in a `<Provider>`.

```typescript
// src/hooks/__tests__/useStock.test.tsx

// 1. Create a real store instance (or a mock one)
const store = configureStore({ reducer: { stock: stockReducer, ... } });

// 2. Define the wrapper
const wrapper = ({ children }) => (
  <Provider store={store}>{children}</Provider>
);

// 3. Render the hook with the wrapper
const { result } = renderHook(() => useStock(), { wrapper });

// 4. Assert initial state
expect(result.current.symbol).toBeNull();
```

### Mocking RTK Query
For API calls, we often mock the *hook* generated by RTK Query rather than the network. This keeps tests fast and deterministic.

```typescript
vi.mock('@/store/api/stockApi', () => ({
  useGetStockDataQuery: vi.fn(), 
}));
```

## 2. Integration Testing Components

When testing components that receive props (like `PriceTicker`), we want to test how they render *different states*.

### Dynamic Props Testing
Does the component change color when the price drops?

```typescript
it('renders negative change with red color', () => {
    const negativeQuote = { ...mockQuote, change: -5.00 };
    render(<PriceTicker quote={negativeQuote} isLoading={false} />);
    
    const element = screen.getByText('-5.00');
    expect(element.className).toContain('text-red-400');
});
```

## 3. The "Testing Pyramid" in Practice

| Level | What we tested | File Example |
| :--- | :--- | :--- |
| **Unit** | Algo Math | `technicalAnalysis.test.ts` |
| **Unit** | Reducer Logic | `stockSlice.test.ts` |
| **Integration** | Hook + Store | `useStock.test.tsx` |
| **Integration** | Component + Props | `PriceTicker.test.tsx` |

## 4. Common Pitfalls
1.  **Mocking too much:** If you mock everything, you aren't testing anything. Try to use real logic where possible (like real reducers).
2.  **Snapshot overuse:** Snapshots break easily. Test precise values (e.g. `getByText('$150.00')`) instead.
3.  **Ignoring cleanup:** `vi.clearAllMocks()` in `beforeEach` is essential to prevent tests from leaking state.
